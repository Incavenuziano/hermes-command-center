# Security Policy

If you discover a security issue in Hermes Command Center, do not open a public issue with exploit details.

Until a dedicated reporting workflow is published, report privately to the project maintainer through the configured repository security channel or a private contact path documented in release materials.

Core security commitments:
- local-first, loopback-by-default posture
- no telemetry by default
- no plaintext secret exposure in logs or API responses
- high-risk actions must be auditable
